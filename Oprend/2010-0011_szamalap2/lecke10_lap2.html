Memóriakezelés, alapvető módszerek, virtuális memória, szegmentálás és megvalósításuk a mai processzorokban

A felhasználói szintű memóriakezelés gépi kódú környezetben

Az utasítások működéséhez szükségünk lesz arra, hogy megismerkedjünk a címképzési módszerekkel. A címkézési művelet azt jelenti, hogy az operandusból előállítjuk a konkrét adatot. A különböző típusokat rövid programrészletekkel szemléltetjük, melyekben egyelőre csak az adatmozgató utasítás lesz a lényeges. Az adatmozgató utasításunk

mov op1, op2

alakú lesz, mely az op1:=op2 műveletet jelenti. Processzorunk – az egyszerűség kedvéért, és mert a lényeget nem befolyásolja – legyen úgynevezett egycímes, ami azt jelenti, hogy az op1 és op2 operandusok (a műveletben szereplő adatok) közül csak az egyik lehet memóriacím. Nézzük meg egy egyszerű program fordítási listáját!

0000 adat segment ;az adatszegmens ;kezdete 0000 000C valami dw 12 ;valami:=12 0002 44 4F 4C 47 4F 5A 41 szoveg db 'DOLGOZAT ! ' ;szöveg, ;karaktersorozat ;54 20 21 20 000B hossz equ ($-szoveg) ;a szöveg hossza 000D adat ends 0000 program segment ;a program szegmens ;kezdete assume cs:program, ds:adat 0000 start: 0000 B9 0000 mov cx, 0 ;cx:=0 0003 BA 184F mov dx, 184fh ;dx:=184fh, 0006 B4 4C mov ah,4ch ;visszatérés a ;DOS-ba 0008 CD 21 int 21h 000A program ends ;kódszegmens vége end start ;a program fizikai ;vége

A szegmens kezdőcíméhez képesti eltolás hexadecimálisan.

A szegmens utasításainak lefordított alakja hexadecimálisan.

A további részek a program szöveges része, ami tartalmaz adminisztrációs utasításokat, címkéket, végrehajtható utasításokat, valamint a ; (pontosvessző) után megjegyzéseket, ami célszerűen az utasítás szemantikájára, végrehajtására utal.

Mielőtt rátérnénk a címzési módszerek tárgyalására, elemezzük processzorunk címképzési módszerét. Abszolút cím alatt egy memóriahelyet fogunk érteni, ahol vagy egy utasítás, vagy egy adat kezdődhet. Ennek kezdőértéke a használható memória kezdőhelye, a végértéke pedig a használható memóriaterület utolsó bájtjának a helye. A relatív cím az adott szegmens kezdetéhez való bájtokban mért eltolást jelenti. Ezt az adott adat vagy utasítás offszetjének – a kezdethez való eltolásnak – nevezik, amit 16 biten ábrázolunk Mivel az első ilyen típusú processzorok még csak egy megabájtnyi memória címzésével kalkuláltak, ezért az abszolút cím kiszámítása a következő módon történik (példaként a kódszegmens-regisztert hívjuk segítségül, ami a program betöltésekor kap kezdőértéket).

Számítsuk ki a mintaprogram harmadik utasításának abszolút címét! Ennek offszetje a hexadecimális 0006, a CS értéke pedig legyen a hexadecimális 44F0 (ez egy kitalált érték, de akár ennyi is lehetne). Az abszolút címet a következőképpen kapjuk: 4*CS+offset, írjuk ki bitenként:

CS: 01000100 11110000

4*CS. 0100 01001111 00000000

offset: 00000000 00000110

________________________________________________________________

Az abszolút cím így: 0100 01001111 00000110, ami hexadecimálisan 04 4F 06.

Mivel ebben az esetben az offszet 16 bites lehet, az egyes szegmensek maximális hossza 216 lehet. Egy ilyen hosszúságú egységet neve lap vagy szakszóval page. Most már megvizsgálhatjuk a címképzési technikákat.

Az operandusok címképzési módszerei alapvetően két nagy csoportba oszthatók, az egyiket direkt címzésnek, a másikat indirekt címzésnek fogjuk nevezni. Mindkettő további módokat tartalmaz. Vegyük sorba ezeket!

  • Direkt címzési módszerek: az operandus közvetlenül a használandó adat címét tartalmazza.
    • Konstans címzés: az operandus egy konstans érték, amely közvetlenül bekerül az utasításkódba.
    • 0000 start:
      0000 B9 002E mov cx, 46 ; számkonstans
      0003 BA 616C mov dx,'al' ; szövegkonstans
      0006 B4 4C mov ah,4ch
      0008 CD 21 int 21h
      000A program ends
      end start

      Ekkor, mint látható, a megjelölt utasításokba bekerült az első esetben a szám konstans hexadecimális értéke: 2E, míg a második esetben a két karakter ASCII kódja hexadecimálisan: 61, ami a kis „a” betű kódja, illetve 6C, ami a kis „l” betű kódja.
    • Regiszter-regiszter címzés. Ebben az esetben a mov utasítás mindkét operandusa regiszter.

    • 0000 start:
      0000 B9 002E mov cx, 46
      0003 8B D1 mov dx,cx ; dx:=cx
      0005 8B D9 mov bx,cx ; bx:=cx
      0007 B4 4C mov ah,4ch
      0009 CD 21 int 21h
      000B program ends
      end start
      Itt a megjelölt utasításokban a kódba van beépítve a két regiszter. Látható, hogy a két azonos típusú utasításban a második bájtban van különbség, ami a két regiszterre utal.
    • Közvetlen címzés. Ilyenkor az egyik operandus egy szimbolikus memóriacím vagy annak egyszerű módosítása, például valamilyen konstans hozzáadása vagy kivonása.

      0000 start:
      0000 8A 1E 0002 R mov bl,szoveg ; bl:=szoveg
      ; első karaktere
      0004 8B 16 0000 R mov dx,szoveg-2 ; dx:=valami
      0008 B8 000B mov ax,hossz ; ax:=hossz
      000B B4 4C mov ah,4ch
      000D CD 21 int 21h
      000F program ends
      end start
      0000 start:
      end start

      Mint az első és második megjelölt utasításban látható, ilyenkor a harmadik és a negyedik bájt az adatszegmens kezdetéhez való eltolást jelenti bájtokban mérve. Az R betű jelenti, hogy a program futtatásakor kerül ide az abszolút cím. A harmadik megjelölt utasításban ez nem egy összetett adat – a szöveg azért összetett, mert egy bájtsorozat –, pontosan két bájtból áll. Ilyenkor nincs szükség külön címkiszámításra, az operandusmezőbe rögtön bekerülhet az adat offszettje.
    • Indirekt címzési módszerek. Ezekben az esetekben az operandus nem közvetlenül az adatot tartalmazza, hanem annak elérhetőségét, tehát a címkiszámítás valamilyen algoritmus alapján történik. Ennek megfelelően a kiszámítás módja alapján a módszereket további csoportokba oszthatjuk. Ezt a fajta címzést formálisan a szögletes zárójelpár – [] – fogja jelölni.
  • Bázisrelatív címzés: címkiszámításra a BX bázisregisztert használjuk

    0000 start:
    0000 BB 0002 R mov bx,offset szoveg ;bx:=a szöveg relatív
    ; címe
    0003 8A 17 mov dl,[bx] ; dl:=a szöveg első karaktere
    0005 B4 4C mov ah,4ch
    0007 CD 21 int 21h
    0009 program ends

    Az első megjelölt utasítás a bx bázisregiszterbe berakja az adatszegmens szövegadatának a 0-hoz – mint a szegmens kezdetének a címe – való eltolásának értékét, ami a jelen esetben kettő, hiszen fizikailag előtte egy kétbájtos adat szerepel a szegmensben. A bx most egy relatív címet tartalmaz, a második utasítás így a bx címen található adatot, ami most a szöveg első karaktere, azaz a „D” betű, átrakja a dx regiszter alacsonyabb helyi értékű bájtjába, a dl-be.
    • Bázisrelatív címzés eltolással. A bázisregiszter tartalmához hozzáadódik az eltolás értéke, az így keletkezett érték lesz a használandó adat relatív címe.

      0000 start:
      0000 BB 0002 R mov bx,offset szoveg
      0003 8A 57 04 mov dl,4[bx] ; dl:=a szöveg 5.
      ; karaktere
      0006 8A B7 0000 R mov dh,valami[bx] ; dh:=a szöveg 1.
      ; karaktere
      000A B4 4C mov ah,4ch
      000C CD 21 int 21h
      000E program ends
      end start

      Az első megjelölt utasításnál a bx értékéhez hozzáadódik az eltolás értéke, ami az utasítás utolsó bájtjában –04 – meg is jelenik, így dl értéke a második „O” betű lesz. A második megjelölt utasításnál a bx tartalmához hozzáadódik a „valami” offset címe, ami az utasítás utolsó két bájtjában az operandus helyén meg is jelenik; ennek értéke 0000, hiszen ez az adatszegmens első adata.
    • Címzés az indexregiszterrel: ez a változat az 1. pontban leírtaknak megfelelően működik, így minden magyarázat nélkül csak a használt példát mutatom be.

      0000 start:
      0000 BE 0002 R mov si,offset szoveg
      0003 8A 14 mov dl,[si] ; dk:=’D’
      0005 B4 4C mov ah,4ch
      0007 CD 21 int 21h
      0009 program ends
      end start
    • Címzés az indexregiszterrel eltolással. Ilyenkor az indexregiszter tartalmához hozzáadódik az eltolás értéke.

      0000 start:
      0000 BE 0002 R mov si,offset szoveg
      0003 8A 54 04 mov dl,4[si] ; dl:=a szöveg 5.
      ; karaktere
      0006 8A B4 0000 R mov dh,valami[si] ; dh:=a szöveg 1.
      ; karaktere
      000A B4 4C mov ah,4ch
      000C CD 21 int 21h
      000E program ends

      end start
      Működése 2. pontban leírtaknak megfelelő, azaz az si – vagy di, hiszen az is indexregiszter – tartalmához hozzáadódik az eltolás értéke, az így keletkezett relatív cím lesz az a használandó adat címe.
    • Indexelt címzés. Működésileg ez felel meg leginkább a magas szintű programozási nyelvekben használatos tömb használatnak.

      0000 start:
      0000 BE 0004 mov si,4 ; si:=4
      0003 8A 94 0002 R mov dl,szoveg[si] ; dl:=a szöveg 5.
      ; karaktere
      0007 B4 4C mov ah,4ch
      0009 CD 21 int 21h
      000B program ends
      end start

      A szöveg a tartalma első karakterének az offszettjét tartalmazza, ami az utasítás 3. és 4. bájtjaiban található, értéke 0002. Az si értéke 4, a kettő összege így 6 lesz, ami a szegmens kezdetéhez képesti 7. bájt, aminek értéke a második „O” betű.
    • Bázisrelatív címzés indexeléssel: a bázisregiszter tartalmához hozzáadódik az indexregiszter tartalma, így keletkezik az a relatív cím, amelyik a használandó adat offsetjét tartalmazza.

      0000 start:
      0000 BB 0002 R mov bx,offset szoveg
      0003 BE 0004 mov si,4 ; si:=4
      0006 8A 10 mov dl,[bx+si] ; dl:=a szöveg 5.
      ; karaktere
      0008 B4 4C mov ah,4ch
      000A CD 21 int 21h
      000C program ends
      end start

      Természetesen indexregiszterként itt is használhatjuk di-t.
    • Bázisrelatív címzés indexeléssel és eltolással: ebben az esetben a három különböző címképző jellemző, a bx bázisregiszter, az si vagy di indexregiszter, valamint az eltolási értékek összeadódnak, melynek eredményeként megkapjuk a használandó adat relatív címét.

      0000 start:
      0000 BB 0000 R mov bx,offset valami
      0003 BE 0004 mov si,4 ; si:=4
      0006 8A 90 0002 R mov dl,szoveg[bx+si] ; dl:=a szöveg 5.
      ; karaktere
      000A 8A 30 mov dh,0[bx+si] ; dh:=a szöveg 3.
      ; karaktere
      000C B4 4C mov ah,4ch
      000E CD 21 int 21h
      0010 program ends
      end start

      Az első megjelölt utasítás eredményeként az si indexregiszter tartalma 4 lesz. A bx tartalma 0, mivel a „valami” szimbolikus címként a szegmens első adata. Ennek megfelelően a második megjelölt utasítás – mivel a „szöveg” offszetje mint az utasításban is látható 2 – relatív címe 2 + 0 + 4, ami a második „O” betű relatív címe. Így dl tartalma a második „O” betű lesz. A harmadik megjelölt utasításnál a címképző jellemzők: a 0 eltolás, bx az előbbiek szerint szintén 0, míg si értéke 4, így a használandó adat offszetje 4, ami az adatszegmensben az „L” betű relatív címe, így dh tartalma az utasítás végrehajtása az „L” ASCII kódja lesz, ami –az adatszegmensből kiolvashatóan – hexadecimális 4C, decimálisan 78.

A szülő- és a gyermekfolyamatok

Példák a szülő- és a gyermekfolyamatokra egy C programozási nyelven írt programban

Feladat

Feladat: a szülő- és a gyermekfolyamat írja ki az azonosítóját, és a szülőfolyamat várja meg a gyermekfolyamat befejeződését (wait függvény használatával)!

Legyen a feladatunk az, hogy készítsünk C programozási nyelven írt programot, amelyik létrehoz egy gyermekfolyamatot. Mindkét folyamat írja ki a saját PID számát a standard kimenetre (azaz a stdout-ra), és a szülőfolyamat várja be a gyermekfolyamat befejeződését. A szülő vizsgálja meg, hogy a gyermekfolyamat normál kilépéssel fejeződött-e be. A megoldáshoz használjuk a wait függvényt.

//-------------------- // Wait2.c //-------------------- #include <stdlib.h> #include <stdio.h> #include <unistd.h> // Például a fork. #include <sys/types.h> // Például a pid_t típus. #include <sys/wait.h> // A wait és a waitpid függvény használatához. int main() { // Az unistd.h header fájlt be kell szerkeszteni. int status; // A wait függvényben a gyermekfolyamat állapota. pid_t child = fork(); // Itt keletkezik a gyermekfolyamat, amelyik a szülőfolyamat // másolataként fogható fel. // A child változóban a keletkezett gyermekfolyamat PID száma lesz // (process identification, vagy más néven a folyamat azonosítója) if (child < 0) { perror("Nem sikerult folyamatot letrehozni\n"); exit(1); } // Kilépünk, mert nem sikerült új gyermekfolyamatot létrehozni. if (child > 0) { printf("En a szulofolyamat vagyok, PID szamom %i \n”, getpid()); // Ezt a kódrészletet csak a szülőfolyamat hajtja végre, // hiszen látja a child változó értékét. wait(&status); // Várakozás a gyermekfolyamat befejeződésére. if (WIFEXITED(status)) { printf("Normal befejezodes\n"); } else { printf("Hibas befejezodes\n"); } printf("Szülő vége\n"); // Mindig ez kerül utoljára kiírásra. } if (child == 0) { printf("En a gyermekfolyamat vagyok, PID szamom %i \n", getpid()); sleep(10); // Ezt a kódrészletet csak a gyermekfolyamat hajtja végre, hiszen // a gyermekfolyamat nem látja a child változó értékét. } return 0; } //--------------------

A futtatás eredménye

>gcc Wait2.c –o Wait2 >./Wait2 En a gyermekfolyamat vagyok, PID szamom 16521 En a szulofolyamat vagyok, PID szamom 16520 Normal befejezodes Szülő vége >./Wait2 & En a szulofolyamat vagyok, PID szamom 12320 En a gyermekfolyamat vagyok, PID szamom 12321 >kill 12321 Hibas befejezodes Szülő vége >

Megfigyelés

A programot többször lefuttatva megfigyelhetjük, hogy a gyermek- és szülőfolyamatnak mindig különböző PID száma van. A normál befejezéshez elég megvárnunk a sleep lejártát, ha azonban a programot a háttérben indítjuk, a gyermekfolyamatot a kill parancs segítségével „kilőhetjük”. A befejeződés mikéntjét ki tudtuk olvasni!

Jóval több lehetőségünk adódik a waitpid függvény használatával. Ennél a függvénynél paraméterben megadhatjuk, hogy milyen PID számú folyamatra várjon (a wait bármilyen gyermekfolyamat befejezésére reagál, előírhatjuk azt is, hogy várja, vagy esetleg csak olvassa le a folyamat státusát.

Szignálok, a szülő-gyerek viszony gyakorlati szempontból

Szignálok, szülő- és gyermekfolyamatok (minireferencia)

Folyamatokkal kapcsolatos függvények

pid_t fork();

Gyermekfolyamat létrehozása – a gyermek PID számát adja vissza a függvény.

Hiba esetén az errno értékek

EAGAIN

Lehet, hogy később létrehozható a gyermekfolyamat.

ENOMEM

Nincs elég memória.

pid_t getpid();

A folyamat PID számát adja vissza.

pid_t getppid();

A szülőfolyamat PID számát adja vissza.

int daemon(int nochdir,int noclose);

Démon folyamatot hoz létre. Ha a nochdir paraméter nem nulla, akkor a gyökérkönyvtár lesz a munkakönyvtár. A noclose nem nulla, akkor a statndard kimenetek a dev/null-ba irányítódnak át. A visszatérési érték nulla, ha sikeres volt a végrehajtás.

pid_t wait(int *status);

Vár egy gyermekfolyamat befejezésére, és az állapotleírást a „status” változóban helyezi el. Javasolt a waitpid használata.

pid_t waitpid(pid_t pid, int *status, int options);

Vár az első paraméterben megadott folyamat befejezésre.

WAIT_ANY – bármelyikre vár.

Az állapotot a második paraméterbe írja vissza.
A harmadik paraméterben a működését befolyásolja.

WNOHANG – nem várakozik, csak kiolvassa az állapotot.
WUNTRACED – bármilyen folyamatról információt ad vissza.

A függvény visszatérési értéke nulla, ha sikeres volt a végrehajtása.

A „status” kiolvasásához való függvények

int WIFEXITED(int status);

Igaz, ha a folyamat szabályosan lép ki.

int WEXITSTATUS(int status);

Az exit által visszaadott értéket adja.

int WIFSIGNALED(int status);

Igaz, ha a folyamat nem fogadott egy jelzést.

int WTERMSIG(int status);

A jelzés számértéke, ami miatt befejeződött a folyamat.

int WCOREDUMP(int status);

Igaz, ha a folyamat meg lett szakítva.

int WIFSTOPPED(int status);

Igaz, ha a gyermekfolyamat be lett fagyasztva.

int WSTOPSIG(int status);

A jelzés számát adja vissza, ami befagyasztotta a folyamatot.

Hiba esetén az errno értékek

EINTR

Külső jelzés félbeszakította.

ECHILD

Nincs gyermekfolyamat.

EINVAL

A kapcsolók érvénytelenek.

system(const char*);

A függvény végrehajtja a paraméterben megadott parancsot, és visszatér a system utáni sorba.

Az EXEC függvénycsalád

Az adott folyamatot „lecseréli” a paraméterben megadott programra vagy parancsra.

int execl(const char *path, const char *arg0, ... /*, (char *)0 */);

Az első paraméter a parancs neve, utána egy lista, amelyben a parancs neve, paraméterei és a végén egy NULL található. A környezeti változói az eredeti folyamaté maradnak.

int execv(const char *path, char *const argv[]);

Az első paraméter a parancs neve, a második paraméter egy tömb, amelyben a parancs neve, paraméterei és a végén egy NULL található. A környezeti változói az eredeti folyamaté maradnak.

int execle(const char *path, const char *arg0, ... /*, (char *)0, char *const envp[]*/);

Az első paraméter a parancs neve, utána egy lista, amelyben a parancs neve, paraméterei és a végén egy NULL található. A harmadik paraméter egy tömb, amelyben név = érték párok szerepelnek.

int execve(const char *path, char *const argv[], char *const envp[]);

Az első paraméter a parancs neve, a második paraméter egy tömb, amelyben a parancs neve, paraméterei és a végén egy NULL található. A környezeti változók lecserélhetők a harmadik paraméterrel. A harmadik paraméter egy tömb, amelyben név = érték párok szerepelnek.

int execlp(const char *file, const char *arg0, ... /*, (char *)0 */);

Az első paraméter a parancs neve, utána egy lista, amelyben a parancs neve, paraméterei és a végén egy NULL található. A Path értékét figyelembe veszi.

int execvp(const char *file, char *const argv[]);

Az első paraméter a parancs neve, a második paraméter egy tömb, amelyben a parancs neve, paraméterei és a végén egy NULL található. A környezeti változói az eredeti folyamaté maradnak. A Path értékét figyelembe veszi.

Hiba esetén az errno értékek

E2BIG

Túl hosszú az argumentum.

EACCES

Jogosultsági probléma.

EINVAL

Érvénytelen jelzésszám.

ELOOP

A listában körkörös hivatkozás van.

ENAMETOOLLONG

Túl hosszú a path vagy a fájlnév.

ENOENT

Nem létező útvonal vagy fájlra hivatkozás.

ENOTDIR

A könyvtár nem létezik.

ENOEXEC

Nem végrehajtható.

Szignálokkal kapcsolatos függvények

int kill(pid_t pid,int sig);

A paraméterben megadott szignál küldése a kijelölt folyamatnak.

Hiba esetén az errno értékek

EINVAL

Érvénytelen jelzésszám.

EPERM

Nincs joga jelzést küldeni annak a folyamatnak.

ESRCH

Nem létezik a megcélzott folyamat.

int raise(int sig);

Szignál küldése saját folyamatnak – általában a jelzéskezelés után használják. Visszatérési értéke nulla, ha sikeres volt a végrehajtás.

sighandler_t signal(pid_t pid, void (*func)(int)));

A jelzéskezelő bejegyzése.

Előre definiált jelzéskezelők

SIG_IGN

A jelzés ignorálása.

SIG_DFL

A jelzéshez az alapértelmezés szerinti működés beállítása.

unsigned alarm(unsigned second);

Alarm jelzés küldése „second” másodperc múlva Visszatérési értéke nulla, ha sikeres a végrehajtás. Nagyobb nulla érték esetén az előző alarmhoz hátralevő idő.

int pause(void);

Jelzés érkezéséig vár.

int sigaction(int sig, const struct sigaction *newact, struct sigaction *oldact);

Megváltoztatja egy adott jelzéshez tartozó jelzéskezelést.

struct sigaction {

void (*sa_handler)(int);

void (*sa_sigaction)(int, siginfo_t *, void *);

sigset_t sa_mask;

int sa_flags;

void (*sa_restorer)(void);

}

struct siginfo_t {

int si_signo; //Signal number

int si_errno; //An errno value

int si_code; //Signal code

pid_t si_pid; //Sending process ID

uid_t si_uid; //Real user ID of sending process

int si_status; //Exit value or signal

clock_t si_utime; //User time consumed

clock_t si_stime; //System time consumed

sigval_t si_value; //Signal value

int si_int; //POSIX.1b signal

void * si_ptr; //POSIX.1b signal

void * si_addr; //Memory location which caused fault

int si_band; //Band event

int si_fd; //File descriptor

}

int sigaddset(sigset_t *set,int sig);

A jelzéscsoporthoz hozzáad egy új jelzést.

int sigdelset(sigset_t *set,int sig);

A jelzéscsoportból kitöröl egy jelzést.

int sigemptyset(sigset_t *set);

Létrehoz egy üres jelzéscsoportot. Ezt vagy a sigfillset függvényt használni kell, mielőtt a jelzéscsoporttal műveletet végeznénk.

int sigfillset(sigset_t *set);

Létrehoz egy jelzéscsoportot, amely minden jelzést tartalmaz. Ezt vagy a sigemptyset függvényt használni kell, mielőtt a jelzéscsoporttal műveletet végeznénk.

int sigismember(sigset_t *set,int sig);

Benne van-e egy jelzés a jelzéscsoportban? Visszatérési értéke egy, ha a jelzés benne van a csoportban. Hiba esetén beállítja az errno értéket.

int sigprocmask(int hogyan, const sigset_t *newset, sigset_t *oldset);

Végrehajt egy adott parancsot a jelzéscsoporton.

A parancsok:

SIG_BLOCK – a második csoporthoz tartozók hozzáadódnak az első csoporthoz, ezeket is blokkolja.

SIG_UNBLOCK – a második jelzéscsoportba tartozókat eltávolítjuk az első csoportból, tehát a blokkolásukat feloldjuk.

SIG_SETMASK – a második jelzéscsoportban megadottak blokkolása.

Ha a második paraméter NULL, akkor a parancs az első jelzéscsoportra vonatkozik.

int sigpending(sigset_t *set);

A függő folyamatok lekérdezése.

int sigsuspend(const sigset_t *mask);

A pause függvénnyel azonos hatású, de megadható a maszkolható jelzéscsoport.

A szülő- és a gyermekfolyamatok

Példák a szülő- és a gyermekfolyamatokra egy C programozási nyelven írt programban

Feladat

Feladat: kétszer hívjuk meg a fork függvényt!

Legyen a feladatunk az, hogy készítsünk C programozási nyelven írt programot, amelyik kétszer hívja meg a fork függvényt! (Most nem figyelünk a zombi folyamatokra.) Próbáljuk meg előre kitalálni, hogy hány kiírást fogunk látni!

//-------------------- // Forkok.c //-------------------- #include <stdlib.h> #include <stdio.h> #include <unistd.h> // Az unistd.h header fájlt be kell szerkeszteni, // amelyben például a fork függvény szerepel. #include <sys/types.h> // Például a pid_t típus. int main() { fork(); // Itt keletkezik a gyermekfolyamat, amelyik a szülőfolyamat // másolataként fogható fel. fork(); // Ez egy újabb gyermekfolyamatot hoz létre. printf("Ez a %i PID szamu folyamat vege\n", getpid()); // A fork utáni részt a szülő- és a gyermekfolyamat is végrehajtja. return 0; } //--------------------

A futtatás eredménye

>gcc Forkok.c –o Forkok >./Forkok Ez a 12656 PID szamu folyamat vege Ez a 12659 PID szamu folyamat vege Ez a 12658 PID szamu folyamat vege Ez a 12657 PID szamu folyamat vege >

Megfigyelés

Vajon miért futott le négyszer a printf függvény, azaz a kiírás? Gondoljuk végig még egyszer a legelejétől kezdve! Az első fork hívás után lemásolódik a programkód fork utáni része, és a gyermekfolyamatként önálló életet kezd.

A szülőfolyamat:

int main() { fork(); fork(); printf(„..”); return 0; }

A szülőfolyamat

első gyermek

folyamata:

fork(); printf(„..”); return 0; }

A szülő- és a gyermekfolyamatban is maradt egy-egy feldolgozatlan fork hívás, ami most végrehajtódik, így valóban létrejön a négy folyamat.

A szülőfolyamat:

fork(); fork(); printf(„..”); return 0; }

A szülőfolyamat második gyermeke:

.... .... printf(„..”); return 0; }

A szülőfolyamat

első gyermek

folyamata:

fork(); printf(„..”); return 0; }

A szülőfolyamat

első gyermek

folyamatának első

gyermeke:

.... printf(„..”); return 0; }

Természetesen máshogy is megszervezhetjük ennek a három gyermekfolyamatnak a létrehozását úgy, hogy a kód szerkezete is jobban tükrözze az egyes folyamatok egymásba ágyazását, és ne hozzunk létre zombi folyamatot sem.

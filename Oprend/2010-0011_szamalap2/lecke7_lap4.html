A szülő- és a gyermekfolyamatok

Példák a szülő- és a gyermekfolyamatokra egy C programozási nyelven írt programban

Feladat

Feladat: a szülő- és a gyermekfolyamat más-más szöveget írjon ki, és a szülőfolyamat várja meg a gyermekfolyamat befejeződését!

Legyen a feladatunk az, hogy készítsünk C programozási nyelven írt programot, amelyik létrehoz egy gyermekfolyamatot, és a szülőfolyamattal együtt mindketten más-más szöveget írjanak ki a standard kimenetre (azaz a stdout-ra), és a szülőfolyamat várja meg a gyermekfolyamat befejeződését.

//-------------------- // Wait.c //-------------------- #include <stdlib.h> #include <stdio.h> #include <unistd.h> // Például a fork. #include <sys/types.h> // Például a pid_t típus. #include <sys/wait.h> // A wait és a waitpid függvény használatához. int main() { // Az unistd.h header fájlt be kell szerkeszteni. int status; // A wait függvényben a gyermekfolyamat állapota. pid_t child = fork(); // Itt keletkezik a gyermekfolyamat, amelyik a szülőfolyamat // másolataként fogható fel. // A child változóban a keletkezett gyermekfolyamat PID száma lesz // (process identification, vagy más néven folyamat azonosítója) if (child < 0) { perror("Nem sikerult folyamatot letrehozni\n"); exit(1); } // Kilépünk, mert nem sikerült új gyermekfolyamatot létrehozni. if (child > 0) { printf("En a szulofolyamat vagyok\n"); // Ezt a kódrészletet csak a szülőfolyamat hajtja végre, hiszen // látja a child változó értékét, amit az általa meghívott fork // adott vissza a számára. wait(&status); printf("Szülő vége\n"); // Mindig ez kerül utoljára kiírásra. } if (child == 0) { printf("En a gyermekfolyamat vagyok\n"); // Ezt a kódrészletet csak a gyermekfolyamat hajtja végre, // hiszen a gyermekfolyamat nem látja a child változó értékét. } return 0; } //--------------------

A futtatás eredménye

>gcc Wait.c –o Wait >./Wait En a szulofolyamat vagyok En a gyermekfolyamat vagyok Szülő vége >

Megfigyelés

Többször lefuttatva a programot, bármelyik folyamat kezdett el futni, mindenképpen a szülőfolyamat fejezi be – az utolsó kiírás mindig a „Szülő vége”.

A fenti kódban – a wait függvény paramétereként – használtuk a status változót, de nem tudjuk, hogy ez pontosan mit is jelent, mit tudhatunk meg az értékéből. Az operációs rendszer ebben a változóban adja vissza, hogy az adott folyamat milyen állapotban fejeződött be. A status változó értékeinek jelentését néhány függvény segítségével értelmezhetjük. A legegyszerűbb eset az, ha a gyermekfolyamat hiba nélkül helyesen lefut. Ekkor a status változót felhasználva a WIFEXITED függvénnyel ezt lekérdezhetjük. (További lehetőségek a mini referenciában olvashatóak.) A folyamatokból hozzáférhetünk a PID számukhoz a getpid függvénnyel.
